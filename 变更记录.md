## v0.1 - 2026-06-07 18:30

### 变更摘要
- 创建可打开的 SwiftUI iOS App 工程 `ioshealth.xcodeproj`。
- 接入 HealthKit 历史数据读取，覆盖心率、步数、活动能量、HRV、血氧、呼吸频率、睡眠、锻炼时长。
- 实现 4 小时粒度预处理、缺失处理、训练集标准化、252 窗口构建和时间顺序切分。
- 实现端侧个人重建模型、多人预测先验、个人预测微调、分位数融合评分和本地模型存储。
- 从当前 PMData/RAIS 公开数据生成 App 内置 `PopulationPriors.json` 多人预测先验。

### 涉及文件
- `ioshealth.xcodeproj/project.pbxproj`
- `ioshealth/HealthAnomalyApp.swift`
- `ioshealth/HealthKitPipeline.swift`
- `ioshealth/Preprocessing.swift`
- `ioshealth/ModelsAndScoring.swift`
- `ioshealth/Storage.swift`
- `ioshealth/Resources/PopulationPriors.json`
- `ioshealth/ioshealth.entitlements`
- `README.md`

### 原因
- 将健康异常检测从 Python 原型推进为可在 iOS 端侧运行的 App 工程。

### 如何部署
- 在 macOS 上执行 `open ioshealth.xcodeproj`。
- 在 Xcode 中修改 `PRODUCT_BUNDLE_IDENTIFIER` 为你的开发者账号可用 ID。
- 给 target 开启 HealthKit capability，并使用真机运行。

## v0.2 - 2026-06-07 18:45

### 变更摘要
- 修复 Swift 代码中 `Double` 与整数字面量混用导致的潜在编译问题。
- 修复 HealthKit async continuation 泛型推断问题。

## v0.3 - 2026-06-07 18:38

### 变更摘要
- 修复测试集切片边界，改用 `valEnd..<windows.count`，避免 Swift 闭区间在空测试集边界下产生风险。
- 新增无第三方依赖的静态校验脚本 `tools/validate_ioshealth.py`。
- 校验脚本覆盖 Xcode 工程关键引用、Swift 括号平衡、坏路径标记、内置先验 JSON 结构和 8 维 HealthKit 先验存在性。
- 在 README 中补充 Windows 本地静态校验入口和 Xcode 真机验证限制。

### 涉及文件
- `ioshealth/Preprocessing.swift`
- `tools/validate_ioshealth.py`
- `README.md`
- `变更记录.md`

### 原因
- Windows 当前无法运行 Xcode 编译，需要把能在本地自动验证的工程完整性检查固化下来，降低迁移到 macOS 编译前的结构性错误。

### 如何部署
- 可在当前目录执行 `python tools/validate_ioshealth.py` 做静态校验。
- 真机部署仍需在 macOS 上用 Xcode 打开 `ioshealth.xcodeproj`，配置开发团队、Bundle ID 和 HealthKit capability。

## v0.4 - 2026-06-07 18:47

### 变更摘要
- 修复 HealthKit 长跨度样本入桶逻辑：睡眠、步数、活动能量、锻炼时长等跨多个 4 小时桶的样本会按重叠时间分配，不再只落到开始时间所在桶。
- 将时间轴终点改为所有样本 `end/start` 的最大值，避免长睡眠样本尾段被截断。
- 收紧 HealthKit 授权与 sample type 转换的 Swift 类型写法，减少 Xcode 编译时的类型推断风险。
- 本地存储从单独模型改为 `StoredModelSnapshot`，重启后恢复模型、数据摘要和最近检测报告。
- 修复分位数排名的等值分数问题，使用 tie-aware 中位秩，避免正常等值分数被直接算到 100 分位。
- 升级 `tools/validate_ioshealth.py`，新增跨桶入桶、HealthKit 类型、分位数排名等关键实现标记检查。

### 涉及文件
- `ioshealth/HealthKitPipeline.swift`
- `ioshealth/Preprocessing.swift`
- `ioshealth/ModelsAndScoring.swift`
- `ioshealth/Storage.swift`
- `tools/validate_ioshealth.py`
- `变更记录.md`

### 原因
- 提升首次导入 Apple Health 后的数据真实性、评分可靠性和 App 重启后的可用性。

### 如何部署
- 运行 `python tools/validate_ioshealth.py` 做本地静态校验。
- 真机部署仍需在 macOS 上用 Xcode 打开 `ioshealth.xcodeproj`，配置开发团队、Bundle ID 和 HealthKit capability 后编译运行。

## v0.5 - 2026-06-07 18:49

### 变更摘要
- 将 HealthKit 历史数据预处理移到 `Task.detached(priority: .userInitiated)` 后台执行，避免首训时阻塞 SwiftUI 主线程。
- 将个人重建模型训练、个人预测微调、最近窗口评分和模型快照保存移到后台执行。
- 新增 `TrainingOutput` 作为 Sendable 训练结果载体，主线程只接收最终模型、摘要和报告。
- 升级 `tools/validate_ioshealth.py`，检查预处理和训练是否都已从 MainActor 卸载。

### 涉及文件
- `ioshealth/HealthAnomalyApp.swift`
- `tools/validate_ioshealth.py`
- `变更记录.md`

### 原因
- 提升真实 Apple Health 大历史导入时的 App 响应性，降低首次启动训练卡住界面的风险。

### 如何部署
- 运行 `python tools/validate_ioshealth.py` 做本地静态校验。
- 真机部署仍需在 macOS 上用 Xcode 编译运行，并实际授权 HealthKit 验证后台训练体验。
